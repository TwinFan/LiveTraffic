v4

v4.3.0

Update: In case of doubt you can always just copy all files from the archive over the files of your existing installation.

At least copy the following files, which have changed compared to v4.2.1:

  • lin|mac|win_x64/LiveTraffic.xpl

Change log:

  • Added AutoATC as a new channel to follow AutoATC's generated AI planes.
    Note: As always, there is a latency as per the Buffering Period, which will cause a time difference between ATC communication and plane movement. You can reduce the Buffering Period to a minimum of 10s (if also reducing Live Data Refresh to 10s), which will however affect all channels and may in turn cause unrealistic behaviour of live planes.
  • ForeFlight output: Added an option to only output traffic, that is not also a "TCAS Target" to avoid duplicates with X-Plane's network output.
    If you prefer X-Plane to output data to ForeFlight directly (via X-Plane's Networking settings) then X-Plane will output your own plane's position and also up to 63 traffic planes, the so-called TCAS targets. When LiveTraffic has TCAS control, those are the 63 nearest planes provided by LiveTraffic. Using the new "Only non-TCAS target traffic" option in the ForeFlight Output settings prevents LiveTraffic from sending TCAS target traffic to ForeFlight. LiveTraffic will then only send planes that are not currently TCAS targets, ie. planes further away, that X-Plane has now knowledge about.
  • Fixed #287: OpenGlider planes with hidden id, that duplicate planes from other channels, are no longer drawn, avoiding duplicate planes in the sky.
    If you disabled OpenGlider just because of those duplicate planes consider enabling it again. Especially in Europe, OpenGlider adds to the immersion with its small planes, gliders, etc.
  • Fixed #284: Allow a Buffering Period of more than 100s with Realtraffic.
  • Fixed #289: Reduced forced output to Message Window so that you can avoid nearly all output to the screen. However, for beginners still the default of "Info" is the recommended choice.
  • Improved how planes are selected for display when seeing more than the configured limit. Should reduce the effect that most distant planes went into and out of visibility freqeuntly.

At least copy the following files, which have changed compared to v2.60:

  • Resources/Doc8643.txt
  • Resources/FlightModels.prf

Change log:

  • X-Plane 12-specific changes
    • ADDED #59: Support for X-Plane 12's wake turbulence, which is applied to TCAS targets, ie. only if LiveTraffic has TCAS control.
      So watch out...going into the wake of an airliner with your C172 may crash you! For training purposes, you can have X-Plane vizualize the wakes by selecting "View > Show 3-D Flight Mode" from the menu or hit [Ctrl-M] eight times until "Wake Turbulence" appears on the screen.

  • NEW #223 option Basics > Cooperation > Hide static objects to remove static objects like towers from all data streams where identifiable.
  • NEW #18 feature: Planes correct angle for wind, crab and de-crab for take-off/landing. The wind considered is taken from current simulator's weather, ie. the same that affects your own aircraft, too.
  • General performance improvements, especially noticeable if many more aircraft are seen in the data than can be displayed.
  • OpenSky: NEW #237 option to enter credentials of a registered user account to allow for a higher number of requests per day (details).
    Remaining requests per day are shown in the OpenSky status in the Status / About page and in the OpenSky settings.
  • ADS-B Exchange: Prepare for version 2 data format (not yet available for the RAPID API interface).
    Provides more data for potentially better altitude and type of vehicle information. Will become mandatory as the v1 format will be phased out by ADS-B Exchange at some point in time.
  • RealTraffic: NEW #229 support for trafic info format "RTTFC" introduced with RT v9, broadcasted on port 49005, becoming the default listening port. Older formats continue to work, too, no matter the network port.
    The network port LiveTraffic listens to is now configurable in Settings > Input Channels > RealTraffic.
  • The Export functionality is also extended to support RealTraffic's RTTFC format: There is a new setting Advanced > Export > Export file format to choose between the previous AITFC and the new RTTFC format.
    Use the updated Resources/SendTraffic.py to send AITFC or RTTFC data back into LiveTraffic.
  • ADDED more potential Ground Vehicles to FlightModels.prf, thanks to @sawiczja and @larsras.
  • FSCharter channel: FIXED #231 static link to flight board.

Change log:

  • CHANGED TCAS handling to new TCAS override approach that was introduced with X-Plane 11.50 Beta 8.
    • TCAS functionality requires X-Plane 11.50b8 or later. On earlier version TCAS can no longer be activated.
    • Up to 63 TCAS blibs will show on Laminar's instruments.
    • You no longer need to configure AI Aircraft in X-Plane's settings just for LiveTraffic's purposes.
    • Other plugins' TCAS implementations as well as 3rd party plugins currently accessing the classic up to 19 multiplayer dataRefs will first need to adapt to the new approach to benefit from the up to 63 planes.
    • Attention developers, if using LiveTraffic's AI multiplayer data: The previous slotting mechanism is removed, that means that planes can potentially change AI slots with every flight loop cycle. They are sorted by distance. This is because the new TCAS target mechanism offers the sim/cockpit2/tcas/targets/modeS_id[64] dataRef array to keep track of an aircraft's identity.
  • TEMPORARILY DEACTIVATED LiveTraffic's own rendering of the map due to a bug in X-Plane 11.50b9, causing crashes when the location changes. Instead, X-Plane's rendering of the now up to 63 TCAS planes is activated so that you see at least some standard icons.
    Bug is filed with Laminar. Map will come back once that bug is fixed.
  • CHANGED #186 identification of ground vehicles, takes operator and call sign into consideration. Call sign matches configured in FlightModel.prf.
  • FIXED identification of ground vehicles for the purpose of chosing a map icon if car ICAO type is configured different from standard ZZZC.
  • FIXED XPMP2 #9: Allowing models with same name from different CSL packages. Matching will pick randomly, also across packages.
    This removes many "Duplicate model" warnings from the log. It remains a warning if the same name is reused within one package.
  • FIXED #183 ghost planes: Some CSL models were not drawn due to a bug while reading xsb_aircraft.txt.
  • FIXED #185 garbage character read from Settings text fields.
  • FIXED #184: Aircraft on the ground, for which no updates are available any longer, were not properly removed but could just sit there forever.
  • REMOVED handling of ICAO/AIRLINE lines from CSL2XSB.py, which previously was meant to define default models per aircraft type and airline. With XPMP2's random pick mechanism this is no longer needed and would even reduce the match quality of the models previously chosen as defaults.
    Execute CSL2XSB.py again on your models to benefit from this change.
  • In Beta versions like this, Log Model Matching (Debug Settings) is forced to be activated.

v0.84.20181224

  • FIXED #61 #43 #58: Better handling of mixed channels. Please run with all channels enabled. A/c will at any point in time be fed by one channel only, but switching to another is possible if (a) the original channel stopped feeding and (b) the new position is in a reasonable direction to prevent 180 degree turns.
    As channels aren't time-synched LT needs to make up for the reported time difference with unusual slow/fast plane movements.
    A channel switch will be document with a DEBUG-level Log.txt entry like DEBUG AA73C3 (SWA1730): SWITCHED CHANNEL from 'ADSB Exchange Live Online' to 'OpenSky Live Online' .
